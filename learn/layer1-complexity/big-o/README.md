# Layer 1 · 점근 표기법 — Big-O·Ω·Θ와 최악/평균/최선

**Notion deep-dive**: https://www.notion.so/3993bd6b405d817d9b75e2770b2f2578

이 개념이 답하는 질문 — *"O(n²)와 O(n log n)의 차이를 '느낌'이 아니라 실측 증가율로 보일 수 있는가, 그리고 O/Ω/Θ가 최악·평균·최선과 왜 다른 축인지 설명할 수 있는가"*

이 개념은 자료구조가 아니라 **복잡도 이론**이라 "직접 구현"이 아닌 **실측 벤치마크**로 검증한다. 같은 입력을 n을 키워가며 O(n²) 알고리즘(버블 정렬, 직접 구현)과 O(n log n) 알고리즘(파이썬 내장 `sorted`)에 먹여 실행시간을 재고, n이 2배 될 때 각각 몇 배로 느려지는지 표로 확인한다.

```bash
python3 bench.py   # → 시간표 + OK
```

Big-O는 상수를 버리지만(그래서 두 값의 절대 크기는 별 의미 없다), **증가율(기울기)**은 절대 못 버린다 — 이 벤치는 그 증가율 차이를 눈으로 보여준다. n을 2배로 키울 때 O(n²)는 ~4배, O(n log n)은 거의 2배 남짓만 늘어나는 걸 표에서 직접 확인할 수 있다.
