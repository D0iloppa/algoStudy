# Layer 2 · 링크드리스트 (내장 컬렉션 없이 바닥부터)

**Notion deep-dive**: https://www.notion.so/3993bd6b405d8199bfc2cfc04b928278

이 개념이 답하는 질문 — *"어레이(연속 메모리)와 링크드리스트(포인터 체인)의 삽입/삭제/탐색 비용이 왜 다른지 메모리 구조로 설명할 수 있는가"*

두 언어로 구현했다. **Python** 은 로직, **C** 는 포인터 체인이 명시적으로 드러나 메모리 구조를 그대로 보여준다.

```bash
python3 linked_list.py              # → OK
gcc -O2 linked_list.c -o ll && ./ll # → OK
```

| 연산 | 어레이 | 링크드 | 비고 |
| --- | --- | --- | --- |
| 임의 접근 | O(1) | O(n) | 링크드는 주소 계산 불가, head부터 순차 |
| 앞 삽입 | O(n) | O(1) | 링크드는 head 참조만 교체 |
| 뒤 삽입 | O(1)* | O(n) | tail 포인터 유지 시 링크드도 O(1) |
| 중간 삭제 | O(n) | O(n) | 링크드는 찾기 O(n) + 잇기 O(1) |
