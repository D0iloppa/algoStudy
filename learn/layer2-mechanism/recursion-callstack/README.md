# Layer 2 · 재귀와 콜 스택

**Notion deep-dive**: https://www.notion.so/3993bd6b405d81179f78e6b23fb27c8d

이 개념이 답하는 질문 — *"재귀 호출마다 콜 스택에 실제로 무엇이 쌓이는지, 왜 모든 재귀는 명시적 스택을 쓰는 반복문으로 바꿀 수 있는지, 그리고 왜 자바·파이썬은 꼬리재귀 최적화(TCO)를 기본 적용하지 않는지 설명할 수 있는가"*

두 언어로 구현했다. **Python** 은 로직, **C** 는 프레임(인자·지역변수·복귀주소)이 콜스택에 쌓이는 구조를 명시적으로 보여준다.

```bash
python3 recursion.py                 # → OK
gcc -O2 recursion.c -o rc && ./rc    # → OK
```

| 형태 | 스택 프레임 | 대표 예 |
| --- | --- | --- |
| 일반 재귀 | 복귀 후 추가 연산이 남아 프레임 유지 필요 | `n * factorial(n-1)` |
| 트리 재귀 | 호출마다 프레임 2개 이상 분기, 중복계산 발생 | `fib(n-1) + fib(n-2)` |
| 꼬리 재귀 | 복귀 후 할 일 없음 → 이론상 프레임 재사용 가능(TCO) | `factorial_tail(n-1, acc*n)` |
| 반복(명시적 스택) | 콜스택 대신 힙/배열에 직접 프레임을 쌓고 pop | `factorial_iter`, `fib_iter` |
