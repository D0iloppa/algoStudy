# Layer 3 · 힙 (우선순위 큐)

**Notion deep-dive**: https://www.notion.so/3993bd6b405d8150ae08de27847c23c1

이 개념이 답하는 질문 — *"힙(우선순위 큐)이 단순 정렬보다 유리해지는 문제 상황을 식별할 수 있는가"*

두 언어로 구현했다. **Python** 은 로직, **C** 는 배열이 곧 완전이진트리라는 사실(포인터 없이 인덱스 산술만으로 부모/자식 관계)을 그대로 드러낸다.

```bash
python3 heap.py                    # → OK
gcc -O2 heap.c -o hp && ./hp       # → OK
```

```
배열: [0, 3, 2, 4, 9, 8, 7, 5, 6]
인덱스: 0   1   2   3   4   5   6   7   8

트리로 보면:
                0
             /     \
            3        2
          /   \     /  \
         4     9   8    7
        / \
       5   6

부모 i → 왼쪽 2i+1, 오른쪽 2i+2 / 자식 i → 부모 (i-1)//2
```

| 연산 | 비용 | 이유 |
| --- | --- | --- |
| push (sift-up) | O(log n) | 완전이진트리 높이가 log n, 최대 log n번 부모와 스왑 |
| pop (sift-down) | O(log n) | 루트를 빼고 맨 끝을 올린 뒤 최대 log n번 자식과 스왑 |
| peek | O(1) | 루트가 항상 최솟값(min-heap) |
| heapify | O(n) | 리프 바로 위부터 역순 sift-down — 전체 정렬 O(n log n)보다 쌈 |

**핵심**: 전체를 정렬할 필요 없이 최솟값/최댓값만 반복 추출하면 될 때(k번째 작은 수, 다익스트라 우선순위 큐, 실시간 스트림 최댓값 유지) 힙이 정렬보다 유리하다.
